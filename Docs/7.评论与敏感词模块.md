# 评论与敏感词模块

最后更新：2026-06-03

## 一、模块范围

本模块负责：

- 文章详情底部评论区。
- 评论回复。
- 评论隐藏、恢复、删除。
- 评论数量维护。
- 评论公开读取安全边界。
- 评论反垃圾基础策略。
- 敏感词库管理。
- 敏感词全站文本写入拦截。

## 二、数据模型

| 表 | 说明 |
| :--- | :--- |
| `article_comment` | 文章评论和回复 |
| `comment_sensitive_word` | 敏感词库 |

评论状态：

- `NORMAL`：正常。
- `HIDDEN`：隐藏。
- `DELETED`：删除。

敏感词字段：

- `word`：敏感词。
- `match_type`：匹配方式，当前主要使用 `CONTAINS`。
- `enabled`：启用状态。
- `deleted`：逻辑删除。

## 三、接口

评论：

| 接口 | 方法 | 说明 |
| :--- | :--- | :--- |
| `/api/article/comment/list` | GET | 评论列表，可匿名读取公开可访问文章评论 |
| `/api/article/comment` | POST | 新增评论或回复，需登录 |
| `/api/article/comment/{id}/hide` | PATCH | 隐藏评论 |
| `/api/article/comment/{id}/restore` | PATCH | 恢复评论 |
| `/api/article/comment/{id}` | DELETE | 删除评论 |

敏感词：

| 接口 | 方法 | 说明 |
| :--- | :--- | :--- |
| `/api/system/sensitive-words` | GET | 敏感词分页 |
| `/api/system/sensitive-words` | POST | 新增敏感词 |
| `/api/system/sensitive-words/{id}` | PUT | 编辑敏感词 |
| `/api/system/sensitive-words/{id}/status` | PATCH | 启用/禁用 |
| `/api/system/sensitive-words/{id}` | DELETE | 删除敏感词 |

兼容旧路径：

```text
/api/article/comment/sensitive-words
```

## 五、评论规则

- 评论内容纯文本展示，不渲染富 HTML。
- 评论内容最多 500 字。
- 评论列表公开读取必须跟随文章可访问性。
- 游客只能读取已发布且可见文章的评论。
- 新增评论和回复需要登录。
- 父评论必须属于同一文章。
- 已删除评论不可继续回复。
- 删除评论只删除当前节点，子回复保留历史记录。
- 隐藏和删除评论必须后端脱敏。
- 公开响应不返回内部 `userId`，使用 `mine` 表示是否当前用户自己的评论。
- 评论数量使用数据库原子增减。

## 六、反垃圾与敏感词规则

评论反垃圾：

- 同一用户 60 秒内最多发布 5 条评论或回复。
- 同一用户同一文章 10 分钟内不能重复提交完全相同内容。
- 敏感词命中时拒绝保存，并返回命中词列表。

敏感词全站拦截范围：

- 文章标题、摘要、正文。
- 评论内容。
- 用户信息。
- 组织、角色、配置、菜单等系统管理文本字段。
- 注册用户名。
- 后续社交私信内容。

实现约定：

- 使用通用 `SensitiveWordGuard`。
- 敏感词管理新增、编辑、启停、删除后应影响后续校验。
- 评论发布路径遇到敏感词表缺失等 SQL 异常时可降级为空规则，避免普通评论直接 500；管理页保留真实错误暴露，便于发现初始化问题。
