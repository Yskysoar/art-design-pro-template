## 一、记录规则

本文档只记录问题情况。按业务模块划分，每个模块内先放未修复、部分修复、持续关注或设计待实现的问题，再放已修复问题。

每条问题至少包含：

- 问题定位。
- 影响范围。
- 当前状态。
- 修复方式或建议修复方式。
- 验证方式。

状态定义：

| 状态 | 含义 |
| :--- | :--- |
| 待修复 | 已确认问题，尚未处理。 |
| 部分修复 | 已缓解主风险，但仍有后续项。 |
| 持续关注 | 当前可接受，上线前或后续阶段继续复核。 |
| 设计待实现 | 规则已确认，代码尚未实现。 |
| 已修复 | 已完成修复并有验证方式。 |
