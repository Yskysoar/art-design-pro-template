# 系统管理模块

最后更新：2026-06-03

## 一、模块范围

系统管理模块负责后台基础管理能力：

- 用户管理。
- 角色管理。
- 组织管理。
- 配置管理。
- 菜单管理。
- 权限码和按钮权限。
- 角色数据权限。

敏感词库页面虽然挂在系统管理菜单中，但业务细节归档到 `7.评论与敏感词模块.md`。

## 二、菜单结构

侧边栏顶层顺序：

1. 仪表盘。
2. 文章管理。
3. 系统管理。

系统管理子菜单顺序：

1. 用户管理。
2. 角色管理。
3. 组织管理。
4. 配置管理。
5. 菜单管理。
6. 敏感词库。

个性设置不进入侧边栏，保留在顶部栏右上角设置抽屉。

## 三、权限规则

- 后端接口必须进行权限码校验。
- 超级管理员 `R_SUPER` 可跳过普通权限码校验。
- 普通角色必须拥有对应权限码。
- 前端按钮权限来自 `/api/user/info` 的 `buttons` 和菜单 `authList`。

典型权限码：

| 模块 | 权限码示例 |
| :--- | :--- |
| 用户 | `system:user:list`, `system:user:add`, `system:user:edit`, `system:user:delete` |
| 角色 | `system:role:list`, `system:role:add`, `system:role:edit`, `system:role:delete`, `system:role:permission` |
| 组织 | `system:org:list`, `system:org:add`, `system:org:edit`, `system:org:delete` |
| 配置 | `system:config:list`, `system:config:add`, `system:config:edit`, `system:config:delete` |
| 菜单 | `system:menu:list`, `system:menu:add`, `system:menu:edit`, `system:menu:delete` |

## 六、业务规则

用户：

- 用户状态使用“启用/禁用”。
- `PATCH /api/user/{id}/status` 用于临时启停账号。
- `DELETE /api/user/{id}` 用于逻辑删除。
- 用户可关联多个组织。

组织：

- 组织树按角色数据权限过滤。
- 删除组织前检查子组织、用户组织关系和角色数据权限关系。

配置：

- 配置键创建后不允许修改。
- 系统内置配置不允许删除。
- 不可编辑配置不允许更新和删除。
- 普通扩展配置逻辑删除后允许重建同名配置键。

菜单：

- 后端返回前端可注册的路由结构。
- 隐藏路由也必须进入菜单数据或静态路由配置。
- 删除菜单需考虑子菜单和角色关联。
