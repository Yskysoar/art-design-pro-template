# 验证码和认证流程

## 一、概述

本文档说明系统图形验证码（Captcha）的后端实现、前端集成方式、公开路径与 JWT 过滤器关系、以及涉及认证流程（登录、注册、重置密码、修改密码）的验证码校验逻辑。

## 二、涉及页面

| 页面 | 路由 | 验证码场景 | 是否需要登录 |
| :--- | :--- | :--- | :--- |
| 登录 | `/auth/login` | 登录时验证 | 否（公开） |
| 注册 | `/auth/register` | 注册时验证 | 否（公开） |
| 忘记密码 | `/auth/forget-password` | 重置密码时验证 | 否（公开） |
| 修改密码 | 顶部栏用户菜单弹窗 | 修改密码时验证 | 是 |

## 三、后端实现

### 3.1 验证码生成

- **位置**: `CaptchaServiceImpl.java`
- **技术**: Java AWT `BufferedImage` + `Graphics2D` 绘制随机字符和干扰线
- **存储**: 内存 `ConcurrentHashMap`（`captchaId -> CaptchaEntry`），有效期 5 分钟
- **字符集**: `23456789ABCDEFGHJKLMNPQRSTUVWXYZabcdefghjkmnpqrstuvwxyz`（排除易混淆字符 `0/O/1/l/I`）
- **输出**: Base64 PNG 图片数据 `data:image/png;base64,...`

### 3.2 验证码验证

所有涉及验证码的接口（登录、注册、重置密码、修改密码）均在 `service` 层调用 `captchaService.validateAndConsume(captchaId, captchaCode)`：

1. 检查 `captchaId` 和 `captchaCode` 是否非空
2. 检查验证码是否过期（生成后超过 5 分钟）
3. 检查验证码是否匹配（不区分大小写）
4. 验证通过后从存储中删除（一次性消费）

### 3.3 接口列表

| 方法 | 路径 | 说明 | 权限 |
| :--- | :--- | :--- | :--- |
| GET | `/api/auth/captcha` | 获取验证码图片 | 公开 |
| POST | `/api/auth/login` | 登录（含验证码校验） | 公开 |
| POST | `/api/auth/register` | 注册（含验证码校验） | 公开 |
| POST | `/api/auth/reset-password` | 重置密码（含验证码校验） | 公开 |
| PUT | `/api/user/profile/password` | 修改密码（含验证码校验） | 需登录 |

### 3.4 公开路径与 JWT 过滤器

**问题修复记录（2026-06-02）：**

后端 `JwtAuthenticationFilter` 对携带无效或过期 JWT 的请求会直接返回 401，导致验证码等公开端点也被拦截。

**修复方式**：在 `JwtAuthenticationFilter` 中覆盖 `shouldNotFilter()` 方法，跳过公开路径：

```java
private static final Set<String> PUBLIC_PATHS = Set.of(
    "/api/auth/login", "/api/auth/captcha",
    "/api/auth/register", "/api/auth/reset-password",
    "/api/health", "/api/article/comment/list"
);

@Override
protected boolean shouldNotFilter(HttpServletRequest request) {
    String path = request.getRequestURI();
    if (PUBLIC_PATHS.contains(path)) return true;
    return path.startsWith("/api/portal/public/")
        || path.startsWith("/api/common/files/");
}
```

## 四、前端集成

### 4.1 验证码加载

每个涉及验证码的页面/组件在 `onMounted` 或对话框打开时调用 `loadCaptcha()`：

```typescript
const loadCaptcha = async () => {
  try {
    const captcha = await fetchCaptcha()
    captchaId.value = captcha.captchaId
    captchaImage.value = captcha.imageBase64  // data:image/png;base64,...
    formData.captchaCode = ''
  } catch {
    console.warn('[Page] 验证码加载失败')
    captchaImage.value = ''
  }
}
```

### 4.2 错误处理

每个页面的 `loadCaptcha` 都增加了 try/catch 错误处理，后端不可用时不阻塞页面渲染。验证码区域会回退到显示刷新按钮（"刷新"文字）。

### 4.3 请求参数

所有认证接口都包含 `captchaId` 和 `captchaCode` 字段：

- `POST /api/auth/login`: `{ userName, password, captchaId, captchaCode }`
- `POST /api/auth/register`: `{ userName, password, captchaId, captchaCode }`
- `POST /api/auth/reset-password`: `{ userName, newPassword, captchaId, captchaCode }`
- `PUT /api/user/profile/password`: `{ oldPassword, newPassword, captchaId, captchaCode }`

## 五、测试要点

1. 未登录时访问登录/注册/忘记密码页，验证码应正常显示
2. 提交空验证码应返回 400 错误
3. 提交错误验证码应返回 400 错误
4. 提交过期验证码应返回 400 错误（验证码 5 分钟后过期）
5. 正常流程：获取验证码 → 输入正确的验证码 → 提交成功
6. 携带过期 JWT 访问公开端点不应被拦截
7. 修改密码对话框中的验证码应独立刷新
