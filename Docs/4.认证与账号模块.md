# 认证与账号模块

最后更新：2026-06-04

## 一、模块范围

认证与账号模块负责：

- 登录。
- 注册。
- 找回/重置密码。
- 当前用户资料。
- 当前用户修改密码。
- 图形验证码。
- 顶部栏用户菜单、头像、锁屏和个人中心入口。

## 二、业务规则

### 2.1 登录

- 登录接口使用用户名、密码和验证码。
- 登录成功返回 `token` 和 `refreshToken`，当前开发期两个字段可使用同一 token。
- 前端写入用户状态后跳转 `redirect` 或首页。
- 登录页、错误页或无效路径不能作为有效 `redirect`。

### 2.2 注册

- 注册接口用于开发期普通用户注册。
- 注册请求必须包含用户名、昵称、密码和验证码。
- 用户名用于登录，昵称用于前端展示；昵称进入敏感词校验。
- 注册成功后创建 `R_USER` 普通角色用户。
- 默认绑定 `ORG_TEMPLATE` 组织。
- 注册必须校验验证码。
- 生产上线前需要确认是否开放自助注册。

### 2.3 重置密码

- 当前重置密码是开发期能力。
- 请求需要账号、新密码和验证码。
- 生产上线前必须增加邮箱、短信、一次性链接或人工审核。

### 2.4 修改密码

- 当前用户修改密码接口不接收用户 ID，只修改当前登录用户。
- 必须校验旧密码。
- 新密码不能和旧密码相同。
- 修改成功后前端清理登录态并要求重新登录。

### 2.5 验证码

- 验证码长度 4 位。
- 字符范围为数字和英文字母。
- 校验不区分大小写。
- 验证码一次性消费。
- 当前使用后端内存缓存，默认 5 分钟过期。

## 三、接口

| 接口 | 方法 | 说明 | 登录 |
| :--- | :--- | :--- | :--- |
| `/api/auth/captcha` | GET | 获取验证码 | 否 |
| `/api/auth/login` | POST | 登录 | 否 |
| `/api/auth/register` | POST | 注册 | 否 |
| `/api/auth/reset-password` | POST | 重置密码 | 否 |
| `/api/user/info` | GET | 当前用户信息 | 是 |
| `/api/user/profile/password` | PUT | 当前用户修改密码 | 是 |

## 四、前端入口

| 页面/组件 | 说明 |
| :--- | :--- |
| `src/views/auth/login` | 登录页 |
| `src/views/auth/register` | 注册页 |
| `src/views/auth/forget-password` | 忘记密码页 |
| `src/components/core/layouts/art-header-bar/widget/ArtUserMenu.vue` | 顶部用户菜单和修改密码弹窗 |
| `src/views/user/user-center` | 个人中心 |
| `src/components/core/layouts/art-screen-lock` | 锁屏 |

## 五、数据来源

- 用户账号和资料来自 `sys_user`。
- 用户角色来自 `sys_user_role` 和 `sys_role`。
- 用户组织来自 `sys_user_org` 和 `sys_org`。
- 前端展示当前用户必须优先使用 `/api/user/info` 返回的数据。

## 六、当前状态

- 登录真实后端接口。
- 注册真实后端接口，前端注册页调用真实 `/api/auth/register`。
- 注册页补齐昵称、验证码字段，注册成功后跳转登录页。
- 重置密码开发期接口。
- 登录、注册、重置密码、修改密码验证码。
- 当前用户资料接口。
- 顶部栏真实头像和用户信息渲染。
- 修改密码成功后强制重新登录。
- 请求方法不匹配时返回 405 统一响应，不再作为 500 未处理异常输出堆栈。
