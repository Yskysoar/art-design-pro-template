# 项目文档目录

> 本目录是新版文档草案索引。旧版文档暂不覆盖，待用户手动确认后再删除或替换。

## 文档结构

| 序号 | 文档名称 | 文档职责 |
| :--- | :--- | :--- |
| 1 | 项目说明文档 | 说明项目技术栈、目录结构、底层架构规划、权限认证、路由、数据、Mock 和通用开发约定。 |
| 2 | 开发审查记录 | 按业务模块记录问题定位、影响范围、修复状态、修复方式和验证方式；模块内先未修复，后已修复。 |
| 3 | 项目测试记录 | 记录测试策略、测试用例、测试命令、测试结果和回归记录。 |
| 4 | 认证与账号模块 | 记录登录、注册、验证码、重置密码、修改密码、当前用户资料和顶部用户入口。 |
| 5 | 系统管理模块 | 记录用户、角色、组织、菜单、配置管理、权限码和数据权限。 |
| 6 | 文章与上传模块 | 记录文章分类、文章发布、文章详情、附件、本地上传和富文本上传。 |
| 7 | 评论与敏感词模块 | 记录文章评论、评论管理、安全加固、反垃圾和敏感词库。 |
| 8 | 社交关注与聊天模块 | 记录关注、粉丝、互关、拉黑、私信聊天、消息额度和 WebSocket 后续预留。 |
| 9 | 文件资源管理模块 | 记录统一上传资源台账、引用识别、资源筛选和未引用资源删除。 |
| 10 | 通知与消息中心模块 | 记录通知台账、顶部铃铛、消息中心、关注/私信/评论回复事件。 |
| 11 | 内容治理与举报审核模块 | 记录文章、评论、私信举报台账、审核处理和权限规则。 |

## 维护规则

- `1/2/3` 是长期核心文档，不承载单个业务模块的全部细节。
- `4+` 按当前已开发或正在开发的业务模块递增。
- 尚未开发的业务模块不提前创建独立文档。
- 新增业务模块时，先补模块文档，再同步 `0.项目文档目录.md`。
- 旧版文档完成迁移前保留，不在本轮覆盖。
