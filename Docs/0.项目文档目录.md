## 维护规则

- 项目级协作、目录边界、Git、Mock、安全和验证规则维护在根目录 `AGENTS.md`。
- `1.项目说明文档.md` 只维护项目说明、业务逻辑、后端对接和 API 设计，不放代码审查问题。
- `2.代码审查总结.md` 只维护审查结论、风险分级、证据文件和修复建议。
- 修改代码或接口约定后，应同步更新受影响文档。
- `Info/` 目录不参与文档读取、写入或整理。
