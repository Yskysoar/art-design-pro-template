## 维护规则

- 项目级协作、目录边界、Git、Mock、安全和验证规则维护在根目录 `AGENTS.md`。
- `Code/art-design-pro-main/` 和 `Code/art-design-pro-main.zip` 是原始前端项目参考快照，只能用于对照修复，不作为当前项目代码或文档扫描对象。
- `1.项目说明文档.md` 只维护项目说明、业务逻辑、后端对接和 API 设计，不放代码审查问题。
- `2.代码审查总结.md` 只维护审查结论、风险分级、证据文件和修复建议。
- `3.后端开发设计方案.md` 维护后端技术选型、架构、权限、数据权限、接口、数据库、Mock SQL 和开发阶段规划。
- `4.后端测试设计.md` 维护后端测试分层、测试用例、测试命令和回归验证方案。
- `5.开发审查问题记录.md` 维护开发过程中的模块化问题台账、修复状态、修复方式和验证建议；新增问题应按模块放置，并保持未修复项在前、已修复项在后。
- 修改代码或接口约定后，应同步更新受影响文档。
- `Info/` 目录不参与文档读取、写入或整理。
