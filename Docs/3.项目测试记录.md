## 一、测试策略

测试按层级拆分：

| 层级 | 工具 | 目的 |
| :--- | :--- | :--- |
| Service 单元测试 | JUnit 5 + Mockito + AssertJ | 验证业务规则、边界条件和异常分支。 |
| Controller 测试 | Spring Boot Test + MockMvc + spring-security-test | 验证认证、权限、参数校验和统一响应。 |
| 数据库回归 | MySQL + schema/mock SQL | 验证表结构、Mock 数据和真实接口。 |
| 前端类型检查 | `vue-tsc --noEmit` | 验证 Vue + TypeScript 类型正确。 |
| 前端构建 | `vite build` | 验证生产构建。 |
| 浏览器回归 | Codex 内置浏览器或手工检查 | 验证页面跳转、布局、交互和接口联调。 |

- 导入 UTF-8 SQL 时使用 `cmd /c` 原始重定向，并指定 `--default-character-set=utf8mb4`。
- 不使用 PowerShell `Get-Content | mysql` 管道导入中文 SQL。

- 注册验证码错误失败。
- 重置密码成功。
