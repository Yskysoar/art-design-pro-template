# 文件资源管理模块

最后更新：2026-06-04

## 一、模块范围

文件资源管理模块负责统一管理项目内上传资源台账。

当前已有上传来源：

- 文章封面上传。
- 文章附件上传。
- 富文本图片上传。
- 社交图片上传。
- 社交附件上传。

本模块第一版目标：

- 查看所有未删除文件资源。
- 按文件名、扩展名、MIME、上传人、大小和时间筛选。
- 查看文件预览/访问地址。
- 查看基础引用状态。
- 删除未被业务引用的文件资源。

第一版暂不做：

- 物理文件清理任务。
- 对象存储迁移。
- 文件移动或重命名。
- 文件内容级病毒扫描。
- 在线编辑文件。

## 二、数据来源

当前统一台账来自 `file_resource` 表。

核心字段：

```text
id, original_name, storage_name, storage_path, url, content_type,
extension, size, sha256, storage_type, uploader_id, create_by,
create_time, deleted
```

## 三、引用识别规则

第一版按现有业务表做保守引用判断：

- `article_attachment.file_id` 引用文章附件。
- `article.cover_url` 引用文章封面 URL。
- `article.content_html` 包含文件 URL 时视为富文本引用。
- `social_message.file_id` 引用社交图片或附件。

只要命中任一引用，就不允许删除文件资源台账。

当前 `referenced` 筛选会先按基础条件查询候选资源，计算引用状态后再做分页，确保返回记录和分页总数一致。该实现适合模板和中小规模开发库；如果文件规模扩大，应改为专用统计 SQL 或资源引用快照表，避免引用状态实时统计带来的 N+1 查询。

## 四、接口规划

| 接口 | 方法 | 说明 |
| :--- | :--- | :--- |
| `/api/file-resource/list` | GET | 文件资源分页 |
| `/api/file-resource/{id}` | DELETE | 删除未引用文件资源 |

分页查询参数：

```ts
interface FileResourceSearchParams {
  current?: number
  size?: number
  keyword?: string
  extension?: string
  contentType?: string
  storageType?: string
  uploaderId?: number
  referenced?: boolean
  minSize?: number
  maxSize?: number
  createTimeStart?: string
  createTimeEnd?: string
}
```

返回字段：

```ts
interface FileResourceItem {
  id: number
  originalName: string
  url: string
  contentType: string
  extension: string
  size: number
  sha256: string
  storageType: string
  uploaderId?: number
  uploaderName?: string
  createBy?: string
  createTime: string
  referenced: boolean
  referenceCount: number
}
```

## 五、权限规则

权限码：

| 权限码 | 说明 |
| :--- | :--- |
| `system:file:manage` | 查看文件资源台账、删除未引用资源 |

超级管理员 `R_SUPER` 跳过权限码校验。普通管理员必须拥有对应权限。

## 六、前端入口

菜单挂载：

```text
系统管理 / 文件资源
```

前端页面：

```text
src/views/system/file/index.vue
```

页面能力：

- 搜索筛选。
- 文件列表。
- 图片缩略图预览。
- 文件大小格式化。
- 复制/打开受控 URL。
- 删除未引用资源。
- 已引用资源显示引用数并禁用删除。

## 七、Mock 数据

开发环境资源样例主要来自：

- `mock_002_article.sql`：文章封面、文章附件和评论相关基础样例。
- `mock_004_social.sql`：社交图片、PDF、txt、docx、xlsx 等聊天样例。
- `mock_005_governance.sql`：额外补充 jpg、png、webp、pdf、docx、xlsx、zip、mp4、mp3、txt，以及 `LOCAL/REMOTE` 存储类型、不同上传人、引用和未引用资源。

Mock SQL 中的前端静态图片和外部示例文件统一标记为 `REMOTE`，真实上传到后端磁盘的资源才使用 `LOCAL` 并通过 `/api/common/files/**` 访问，避免文件资源页误判为本地上传文件。

## 八、安全策略

- 删除只做逻辑删除，不删除磁盘文件。
- 已引用文件不允许删除。
- 文件下载和预览仍走 `/api/common/files/**` 受控读取。
- SVG 当前仍不开放上传。
- 生产化阶段再补物理文件清理、资源隔离、病毒扫描或图片重编码。
