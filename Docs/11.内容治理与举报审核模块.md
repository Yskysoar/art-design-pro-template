# 内容治理与举报审核模块

最后更新：2026-06-04

## 一、模块范围

内容治理与举报审核模块用于承接文章、评论、私信的人工治理需求。

第一版已覆盖：

- 用户提交举报。
- 文章详情页提交文章举报。
- 文章评论和回复提交评论举报。
- 社交聊天消息提交私信举报。
- 敏感词库管理入口。
- 后台举报审核列表。
- 按目标类型、原因、状态筛选。
- 管理员处理举报。
- 权限码控制后台审核入口。

第一版暂不做：

- 自动下架文章。
- 自动隐藏评论。
- 自动删除私信。
- 自动封禁用户。
- 机器审核、图片识别或大模型审核。

不做自动处罚的原因：当前业务仍在模板开发阶段，内容治理需要保留人工判断，避免误伤正常内容。

## 二、数据模型

统一举报表：

```text
content_report
```

核心字段：

```text
id, target_type, target_id, reason_type, description,
reporter_id, reporter_name, status, handler_id, handler_name,
handling_remark, handled_time, create_time, update_time, deleted
```

目标类型：

| 类型 | 说明 |
| :--- | :--- |
| `ARTICLE` | 文章 |
| `COMMENT` | 评论 |
| `PRIVATE_MESSAGE` | 私信 |

处理状态：

| 状态 | 说明 |
| :--- | :--- |
| `PENDING` | 待处理 |
| `PROCESSING` | 处理中 |
| `RESOLVED` | 已处理 |
| `REJECTED` | 已驳回 |

## 三、接口

| 接口 | 方法 | 说明 |
| :--- | :--- | :--- |
| `/api/moderation/reports` | POST | 创建举报 |
| `/api/moderation/reports` | GET | 举报审核分页 |
| `/api/moderation/reports/{id}/handle` | PATCH | 处理举报 |

创建举报会校验目标存在：

- 文章：`article.id`
- 评论：`article_comment.id`
- 私信：`social_message.id`，且当前举报人必须是消息发送人或接收人。

举报原因使用后端白名单校验，当前支持：

```text
SPAM, HARASSMENT, PERSONAL_INFO, COPYRIGHT, FRAUD,
MISLEADING, FILE_RISK, OUTDATED, OFF_TOPIC, LOW_QUALITY, OTHER
```

## 四、权限规则

权限码：

| 权限码 | 说明 |
| :--- | :--- |
| `moderation:report:manage` | 查看并处理举报 |

超级管理员拥有全部权限；内容管理员在 Mock 数据中拥有举报审核权限。

## 五、前端入口

菜单：

```text
内容治理 / 举报审核
内容治理 / 敏感词库
```

页面：

```text
src/views/moderation/reports/index.vue
```

页面能力：

- 前台文章详情、评论/回复和私信消息可打开举报弹窗。
- 举报原因使用固定枚举，补充说明最多 500 字。
- 举报列表。
- 目标类型筛选。
- 状态筛选。
- 原因筛选。
- 状态标签展示。
- 处理弹窗。

## 六、Mock 数据

`mock_005_governance.sql` 已补充举报审核调试样例：

- 目标类型覆盖 `ARTICLE`、`COMMENT`、`PRIVATE_MESSAGE`。
- 处理状态覆盖 `PENDING`、`PROCESSING`、`RESOLVED`、`REJECTED`。
- 原因类型覆盖过期内容、骚扰、垃圾私信、版权、个人信息、诈骗、低质量、偏题、文件风险、误导信息等。
- 同时覆盖未处理、处理中有备注、已处理有处理人和处理时间、已驳回等后台列表状态。

## 七、后续计划

- 处理举报时可联动文章下架、评论隐藏、私信删除或账号处罚。
- 增加处理日志表，记录多次流转历史。
- 与通知中心联动，向举报人推送处理结果。
